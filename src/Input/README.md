# Input Module

The input layer translates SDL events into grid camera movement, layout edits, selection workflows (click, marquee, and multi-anchor drag), and editor state updates.

## Files
- `input_handler.h` / `input_handler.c` — central dispatcher invoked from `main.c`. It asks `Global_RebuildHitboxesIfDirty()` for fresh hitboxes before forwarding events to the device-specific handlers and routes `SDL_TEXTINPUT` / mouse motion into the UI layer so save/root text dialogs and the load dropdown can react.
- `input_routing_policy.h` / `input_routing_policy.c` — explicit top-level `IR1` normalize-policy seam for classifying raw events into action class + route policy with text-entry/global-shortcut precedence (`Esc` global only when text capture is inactive).
- `input_mouse.h` / `input_mouse.c` — handles mouse wheel zoom, lasso selection, shift-click additive selection, double-click collapse of selection groups, left-drag group movement (with Alt overriding snapping for the primary anchor), right-click wall placement via `Editor_ClickAt`, drag-to-pan behaviour, and free-view orbit control via `Alt/Option + mouse move` (no mouse button required). In `SPACE_MODE_3D + FREE_VIEW`, clicking a gizmo axis handle starts an axis-constrained drag session with signed motion from drag-start, default grid-step quantization, and `Shift` smooth override; undo is captured once at drag start and the session closes on mouse-up. `LD3D-F4` adds hover/select routing for plane primitive hitboxes (`HITBOX_OBJECT3D`) so authored planes can be selected from the viewport. `LD3D-F4.5` adds direct plane resize drags from viewport handles (`HITBOX_OBJECT3D_PLANE_CORNER` + `HITBOX_OBJECT3D_PLANE_EDGE`) with one undo capture per drag and `Alt` smooth override (`default = snapped`). `LD3D-F5.3` extends prism editing in two lanes: direct handle drags for local `W/H` resize, plus free-view constrained prism-handle gizmo drags (`HITBOX_OBJECT3D_GIZMO_AXIS`) where corner handles allow local `U/V/N` motion, edge handles allow two-axis motion (in-plane normal + `N`), and depth edits route through the dedicated depth-resize layout API. `LD3D-F7` adds object-rotation drag on center gizmo axes when gizmo mode is set to rotate (default snapped to `15°`, `Shift` free rotation), with deterministic baseline-to-angle composition for stable persisted orientation. Wheel zoom consumes high-precision scroll deltas and zooms around the cursor for smoother trackpad/mouse behavior. It also calls `UIPanel_HandleClick` so UI interactions short-circuit layout edits, updates hover highlights, and raises `Global_FlagGridChanged` whenever panning/zooming alters the camera.
- `input_mouse_drag.h` / `mouse/input_mouse_drag.c` — shared mouse-drag session helpers extracted from the main mouse handler. Owns object/anchor drag lifecycle slices, drag-state transitions, and the bounded helper contracts shared across viewport drag modes.
- `input_keyboard.h` / `input_keyboard.c` — polls held keys for continuous panning and zoom, toggles the global quit flag on `Esc`, and reacts to discrete key presses: `M` toggles `SpaceMode` (`2D`/`3D`) with persistence, `X` toggles object center-gizmo mode (`Move`/`Rotate`), `Shift` constrains wall placement (and signals shift state to the editor for marquee activation), `D` toggles delete modes, `1/2/3` switch active edit planes (`XY/YZ/XZ`), `[`/`]` adjust active plane offset, `O` recentres the grid on the selected anchor, `P` toggles anchor persistence, `Shift+P` creates a plane primitive on the active construction plane (`Alt+Shift+P` disables bounds lock for that creation), `Shift+R` creates a rectangular prism primitive (`Alt+Shift+R` disables bounds lock for that creation), `Shift+C` toggles center crosshair visibility, `C` converts anchors between corner/curve, `L` toggles linked handles, and `Delete/Backspace` remove the selected wall, anchor, or selected 3D object. The same object-creation paths are available from right-panel buttons (`Add Plane`, `Add Prism`), selected prism dimensions can be edited via right-panel buttons (`Prism W`, `Prism H`, `Prism D`) using typed dialogs, scene bounds can be toggled/edited from right-panel bounds controls (`Bounds`, `Clamp`, `Edit BMin`, `Edit BMax`), and display units can be cycled from the `Units` button for unit-aware dimension/bounds entry-display. `Ctrl/Cmd+Z` performs undo, `Ctrl/Cmd+Shift+Z` or `Ctrl/Cmd+Y` performs redo, `Ctrl/Cmd+B` (`Shift` variant for output root) opens native root folder choosers, and `Ctrl/Cmd+I` / `Ctrl/Cmd+O` open typed root edit dialogs. The handler defers to the panel when save/root/dimension/bounds dialogs are capturing keyboard input.

## Interactions
- Mouse and keyboard handlers depend on `Global_Get()` to access the grid, layout, and editor. The mouse path now mirrors editor functionality: shift starts marquee selection, Alt overrides snapping during drags, and multi-selection data is stored in `EditorState`.
- During the 3D migration phase, mouse picks are now resolved against the active edit plane using shared ray/plane helpers, then flow through 3D-ready editor/layout APIs.
- `SpaceMode` constraints are enforced in input:
  - `2D` mode locks editing to `XY@z=0` and blocks free-view/plane-switch controls,
  - `3D` mode enables free-view and plane controls.
- Input projection/mapping now flows through `Core/adapters/space_mode_adapter.c` (via `space_mode_adapter.h`) so mode-aware plane/camera behavior is shared with render and hit-test code.
- Curved-handle dragging now derives polar handle values from world-space deltas projected onto the active edit plane, so handle edits remain plane-local and camera-independent.
- In `FREE_VIEW`, Alt/Option orbit updates free-camera yaw/pitch and keeps the orbit target at the layout centroid for fast model inspection.
- While a gizmo drag session is active, orbit is suppressed so axis-constrained editing is not interrupted.
- While a plane/prism-resize drag or prism-gizmo drag is active, orbit is suppressed so handle editing remains stable.
- Overlap resolution now relies on ranked hitbox picking (handles/points over walls, then near-plane and near-cursor tie-breaking), improving selection stability in multi-depth scenes.
- `Global_RebuildHitboxesIfDirty` centralises hitbox regeneration so the input layer never runs stale selection or handle hit-tests.
- Workspace Authoring is routed before focused editor input: `Alt+C` then `Alt+V` toggles authoring through the shared `kit_workspace_authoring` chord helper, `Tab` cycles draft overlay modes, `Enter` applies, and `Esc` cancels active authoring before the global quit route sees the key. In active pane-overlay mode, mouse motion and left clicks are captured by the authoring host so shared overlay button hit testing can drive Mode/Apply/Cancel without leaking clicks into layout editing.
- UI button actions triggered from `UIPanel_HandleClick` call into layout persistence, root-path controls, grid zoom helpers, editor mode toggles, and history capture (raising the relevant dirty flags as needed). Save/root dialogs and the load dropdown consume input so accidental layout edits are avoided while typing or choosing configs.
