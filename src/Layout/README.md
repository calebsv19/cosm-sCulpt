# Layout Module

The layout module stores the floor-plan geometry as anchors connected by walls, supports persistence, and prepares data for rendering and selection.

## Files
- `layout.h` / `layout.c` — define the `Layout`, `Anchor`, and `Wall` structures and implement the top-level management API. Anchors now store 3D world positions (`Vec3 pos`) plus bezier metadata (`AnchorType`, linked handle flag plus polar in/out handles) so curved segments can be rendered and edited. The API exposes both compatibility 2D entry points (`Layout_AddAnchor`, `Layout_AddWall`) and 3D-native entry points (`Layout_AddAnchor3`, `Layout_AddWall3`) for migration. Scene/object helpers are now split into focused source lanes:
  - `scene/layout_scene3d.c` owns scene-3D contracts (`scene3d.bounds`, `scene3d.constructionPlane`) and related clamp/validation helpers. It now exposes scene-bounds face/edge/corner/center handle helpers for handle validity, legal axis masks, handle world points, world-axis vectors, `Layout_ResizeSceneBounds3DFromHandle(...)`, `Layout_TranslateSceneBounds3D(...)`, and `Layout_FitSceneBounds3DToObject(...)` so viewport/manual fit commands mutate bounds through one layout API.
  - `scene/layout_object_store.c` owns `LayoutObjectStore` with stable IDs, tombstone-safe delete behavior, `core_object` dimensional-rule validation, primitive corner extraction, and world AABB extraction for placement/fit commands.
  - `primitives/layout_primitives_create.c` owns primitive authoring contracts (`PlanePrimitiveCreateParams`, `RectPrismPrimitiveCreateParams`, `Layout_CreatePlanePrimitive(...)`, `Layout_CreateRectPrismPrimitive(...)`).
  - `primitives/layout_primitives_resize.c` and `primitives/layout_primitives_rect_prism_resize.c` own plane/prism resize helpers and constrained depth-resize behavior.
  - `layout.c` remains the umbrella layout API and shared topology/editing entrypoint.
  `LD3D-F7` adds orientation editing via `Layout_RotateObject3D(...)`, rotating plane/prism frames around world gizmo axes with explicit orthonormal compose order, optional baseline composition, and lock-to-bounds safety checks. It handles anchor deduplication, wall creation, removal, deletion marking, auto-prune logic driven by the editor's delete mode, bezier handle seeding when constraints allow, and compaction of deleted entities. Mutating functions call `Global_FlagLayoutChanged` so dirty state is tracked centrally.
- `layout_origin.h` / `layout_origin.c` — recentre the layout by shifting all anchors so a chosen anchor becomes the world origin and updating the grid offsets to keep it in view (raising both layout and grid dirty flags).
- `layout_json.h` / `layout_json.c` — load and save layouts using cJSON (`external/cjson`). Persists anchor positions (`x/y/z`), persistence flags, anchor type, linked-handle state, handle basis axis (`handleAxis`), polar handle data, scene-3D bounds (`scene3d.bounds`), construction-plane contract (`scene3d.constructionPlane` with `axis_aligned` or `custom_frame`), and `objects3d` primitive payloads (plane + rect-prism payload contracts) in schema v8. Rebuilds walls with safety checks, emits metadata (`schemaVersion`, `generator`, `gridSize`), refuses to load unsupported schema versions, and logs failures via SDL. Backward compatibility is preserved for older files that only contain `x/y` (loaded as `z = 0`) and omit scene-bounds/construction-plane/object metadata (defaults restored), and additive unknown fields are ignored so new metadata can be introduced without breaking existing loaders. In addition to file IO, `Layout_SaveToString` / `Layout_LoadFromString` expose in-memory snapshots for the undo/redo system. Successful loads mark the layout as dirty so hitboxes rebuild on the next frame.
- `render_layout.h` / `render_layout.c` — draw scene-bounds reference wireframe, optional free-view scene-bounds face/edge/corner/center edit handles, primitive placement previews, plane primitives, prism primitives, walls, and anchors. Plane primitives render as projected quads with deterministic edge/diagonal styling and selected/hovered highlight colours; selected planes include explicit corner/edge resize markers with active-handle highlight. Rect-prism wireframe rendering includes selected/hovered highlight and selected-face resize markers. Placement previews render ghost plane/prism wireframes from `Editor_PrimitivePlacementPreview_Build(...)`, so the preview and creation path use the same spawn policy. Existing anchor/wall behavior remains: selected/hovered anchors (including curved handle gizmos), bezier sampling between endpoint handles, persistent-anchor colouring, and plane-distance attenuation. Projection routing uses the shared mode adapter seam in `Core/adapters/space_mode_adapter.c` instead of raw per-call plane/camera wiring.
- `hitbox_system.h` / `hitbox_system.c` — rebuilds screen-space rectangles covering walls, anchors, bezier handles, object primitives (plane + rect-prism), object resize handles (corners/edges for selected object), plane/prism free-view resize gizmo handles (`HITBOX_OBJECT3D_GIZMO_AXIS`), optional scene-bounds face/edge/corner/center handles plus selected-handle constrained axis hitboxes, and (when active) selected-anchor free-view axis-gizmo handles so the input system can do hit testing. Selected plane corners expose legal local `U/V` gizmo axes, while selected plane edges expose only the one local resize axis. Scene-bounds faces expose one legal world axis, bounds edges expose two axes, bounds corners expose all three axes, and the bounds center exposes all three translate axes when handle visibility is enabled. Object handle-gizmo hitboxes consume the shared app-local `ObjectHandleGizmoTarget` helper so plane/prism target rules stay aligned with render and input drag behavior. Deleted entities are skipped, keeping intermediate states (pre-compaction) safe. Hitboxes are generated using the active edit plane projection, and unconnected anchors still receive hitboxes so they remain selectable after deleting neighbouring walls. Cursor hits are resolved through ranked picking (type priority, view depth, cursor proximity) for stable overlap behavior; ranking prioritizes scene-bounds/object/anchor gizmos before concrete handles, then points/walls/object bodies.

## Concepts
- Anchors track their connected wall indices; during compaction the module rewrites those indices so references stay valid after deletions. For curved anchors, each wall uses the slot that corresponds to its direction (incoming or outgoing), ensuring bezier handles remain stable when the topology changes.
- Connection constraints are now topology-aware: corner anchors can exceed 2 connections, while curve anchors remain limited to 2 so handle semantics stay valid.
- Curved anchors require exactly two connected walls. `Layout_SetAnchorType` checks this constraint, seeds the handles from wall directions in the active edit plane basis, and mirrors/locks handles when requested. The input layer can toggle linkage or edit handles directly, and linked handles remain symmetric thanks to `Anchor_ForceLinkedHandles`.
- Dirty tracking: `Global_FlagLayoutChanged` marks the layout for compaction, which now runs on-demand (via `Global_TickSystems` or the input/render hot path) instead of every frame. `Global_OnLayoutSaved/Loaded` snapshot the layout JSON to keep undo, dirty flags, and the active config path unified.
- `Anchor.isPersistent` prevents auto-prune from discarding a disconnected anchor, letting you pin important points while cleaning stray nodes.

## Related Modules
- `Editor_ClickAt`, multi-anchor dragging, and delete actions rely on `Layout_AddWall`, `Layout_RemoveWall`, `Layout_RemoveAnchor`, and `Layout_SetAnchorType`.
- New 3D migration paths can call `Layout_AddAnchor3` / `Layout_AddWall3` directly, while existing 2D flows continue using compatibility wrappers.
- `Global_RebuildHitboxesIfDirty` schedules compaction and hitbox regeneration after layout mutations so selection stays accurate.
- UI buttons for saving/loading and origin reset call into this module's JSON and origin helpers (raising the appropriate dirty flags along the way).
