# Editor Module

The editor layer manages the wall-placement workflow, bezier anchor editing, multi-anchor selection/dragging, and overlay rendering for in-progress actions.

## Files
- `editor.h` / `editor.c` — define `EditorState`, track the current tool mode, anchor/hover selections, delete mode, shift-state, marquee selection bounds, and multi-selection arrays. Maintain undo/redo stacks (JSON snapshots of the layout), expose `Editor_HistoryCapture`, `Editor_Undo`, `Editor_Redo`, `Editor_ClearHistory`, and implement helpers for selection editing (`Editor_SelectAnchor`, `Editor_SelectAnchorsInBox`, `Editor_BeginAnchorDrag`, etc.). The placement and drag paths are now 3D-ready (`Vec3` world positions), while current interaction behavior remains XY-constrained for compatibility. Projection/plane semantics now flow through `space_mode_adapter` so editor selection and placement use the same mode-aware view context as input/render.
- `render_editor.h` / `render_editor.c` — render helper functions: `Render_Editor_Anchor` shows the starting anchor while placing a wall, `Render_Editor_GhostWall` draws the snapped preview line (respecting shift locking), `Render_Editor_SelectionBox` draws the translucent marquee while lasso selecting, and `Render_Editor_AxisGizmo` draws six directional 3D translate handles around the selected anchor when in `SPACE_MODE_3D + FREE_VIEW` with hover/active visual states. `Render_EditorOverlay` bundles overlay calls. A placeholder exists for future selected-wall highlighting.
- `space_gizmo_drag.h` / `space_gizmo_drag.c` — contracts for free-view 3D axis-gizmo drag math and session state (`+/-X`, `+/-Y`, `+/-Z`). Exposes signed mouse projection, world-distance conversion, quantized-vs-smooth resolution, and axis-distance application helpers used by upcoming gizmo input/hitbox slices.

## Interactions
- `Input_MouseHandle` calls `Editor_ClickAt` on right-clicks with grid-snapped positions, feeds hover information back into `EditorState` (including gizmo-axis hover lane in free-view), and now drives selection workflows (shift-click, marquee, double-click focus, Alt drag). In `SPACE_MODE_3D + FREE_VIEW`, gizmo-handle click starts an axis-constrained drag session that uses signed drag distance, default grid quantization, and `Shift` smooth override before routing through `Editor_UpdateAnchorDrag`. The editor exposes helper APIs so the input layer can add/remove anchors without duplicating array logic.
- `Input_KeyboardHandle` toggles `shiftHeld`, delete modes, origin shifting, pinning, converts anchors between corner/curve modes, toggles linked bezier handles, deletes selected elements, and wires `Ctrl/Cmd + Z` / `Y` into the undo stack.
- UI buttons (load/save JSON, reset origin, pin anchor, link handles, zoom) call `Editor_HistoryCapture` before mutating the layout so operations remain reversible, and invoke `Editor_ClearHistory` when a new config is loaded or saved as a baseline.
- `Render_Frame` invokes the overlay functions so the editor feedback (ghost walls, marquee, drag colouring) appears between layout drawing and UI rendering. The info overlay pulls selection/handle metadata from `EditorState` to display group counts, drag mode, and bezier values.
